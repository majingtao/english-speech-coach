# English Speech Coach 定价与利润测算（年利润目标：10 万元）

## 1. 适用范围

本测算面向当前产品形态：

- 用户：中国大陆英语口语考试学生
- 架构：云端部署，`LLM + ASR + TTS` 全部走 API
- 收费：以订阅（月费）为主，按量作为超额兜底

## 2. 建议定价（用于测算）

- 标准版：`59 元/月`（建议含约 500 分钟）
- 冲刺版：`99 元/月`（建议含约 1000 分钟）
- 超额：`0.18 ~ 0.25 元/分钟`

> 说明：以下利润测算不含人工工资与投放买量，仅计算“业务毛利 - 云基础成本”。

## 3. 核心假设与公式

### 3.1 核心变量

- `ARPPU（月）`：每付费用户月收入
- `API 成本（月/人）`：由 LLM/ASR/TTS 用量决定
- `支付与通道费（月/人）`：按收入比例估算（这里按 2%）
- `固定成本（月）`：服务器、数据库、对象存储、日志监控、带宽等

### 3.2 公式

- 单付费用户月贡献毛利  
  `贡献毛利/月 = ARPPU - API成本 - 通道费`
- 单付费用户年贡献毛利  
  `贡献毛利/年 = 贡献毛利/月 * 12`
- 达到目标利润所需付费用户数  
  `付费用户数 = (目标年利润 + 年固定成本) / 单付费用户年贡献毛利`

## 4. 服务器/硬件成本（云部署估算）

由于推理走外部 API，本地不需要自建 GPU 集群，主要是通用云资源：

- 应用服务（2 台，4C8G）：约 `500 元/月`
- 托管数据库 + Redis：约 `250 元/月`
- 对象存储/CDN/带宽：约 `150 元/月`
- 监控日志/短信/杂项：约 `100 元/月`

合计约 `1000 元/月`，年固定成本约 `1.2 万元`。

## 5. 年利润 10 万所需用户数

目标：年利润 `100,000 元`  
固定成本：年 `12,000 元`  
需覆盖总额：`112,000 元`

### 5.1 基准场景（建议按这个做经营目标）

- ARPPU：`67 元/月`（59/99 套餐有一定混合）
- API 成本：`14 元/月/人`
- 通道费：`1.34 元/月/人`（按 2%）
- 单用户月贡献毛利：`51.66 元`
- 单用户年贡献毛利：`619.92 元`

所需付费用户数：

`112,000 / 619.92 ≈ 181 人`

建议按安全垫执行目标：`200 名稳定付费用户`

### 5.2 保守场景

- ARPPU：`59 元/月`
- API 成本：`20 元/月/人`（使用时长更高）
- 通道费：`1.18 元/月/人`
- 单用户年贡献毛利：`453.84 元`

所需付费用户数：

`112,000 / 453.84 ≈ 247 人`

### 5.3 乐观场景

- ARPPU：`75 元/月`
- API 成本：`11 元/月/人`
- 通道费：`1.50 元/月/人`
- 单用户年贡献毛利：`750 元`

所需付费用户数：

`112,000 / 750 ≈ 150 人`

## 6. 转化到总体用户规模（便于增长规划）

若按付费转化率估算：

- 目标付费用户 `200 人`（基准）
- 付费率 `10%` => 需要 MAU 约 `2,000`
- 付费率 `8%` => 需要 MAU 约 `2,500`
- 付费率 `5%` => 需要 MAU 约 `4,000`

## 7. 执行建议

1. 先以“基准场景”作为经营目标：`200 付费用户`达成 `10 万年利润`。
2. 强控语音成本（尤其 TTS 时长），API 成本每降低 `1 元/月/人`，利润会显著提升。
3. 采用“月费 + 超额按量”，避免重度用户把利润打穿。
4. 每月复盘三项核心指标：`ARPPU`、`人均分钟数`、`API成本/人`，动态校准价格。

## 8. 4 vCPU / 16 GiB 阿里云 ECS 可用性与容量评估

结论先说：**可以用作早期生产**，前提是推理仍走外部 API（不在本机跑大模型）。  
是否够用的核心瓶颈通常不是 CPU/内存，而是**带宽**与**峰值并发**。

### 8.1 适用前提

- 应用层只做鉴权、编排、会话管理、题库读写、API 转发
- LLM/ASR/TTS 计算在外部厂商 API
- 单机部署（无多机负载均衡）

### 8.2 单机容量（经验区间）

在常规 Node/Python Web 服务下，4C16G 大致可承载：

- 普通 API 请求：约 `80 ~ 150 RPS`（保守到基准）
- 实时会话并发（含语音交互转发）：约 `120 ~ 250` 并发用户

> 建议把 `150 并发` 作为安全经营上限，超过后再横向扩容。

### 8.3 换算成“能支撑多少用户”

若学生日均使用 `15 分钟`，且主要集中在放学后 `3 小时`，再乘一个峰值放大系数（约 2）：

- 峰值并发近似：`DAU × 15 / 180 × 2 = DAU / 6`
- 反推 DAU：`DAU ≈ 峰值并发 × 6`

按安全并发 `150` 计算：

- 可支撑 DAU 约 `900`
- 若付费率按 `8% ~ 12%`，对应付费用户约 `72 ~ 108`

按激进并发 `250` 计算：

- 可支撑 DAU 约 `1500`
- 对应付费用户约 `120 ~ 180`

### 8.4 对年利润 10 万目标的影响

前文基准目标为 `200` 名稳定付费用户。  
单台 4C16G 在保守安全线下通常偏紧，建议：

1. 早期先用 1 台起步（快速验证）
2. 达到 `120~150` 付费用户后，准备升配或双机
3. 到 `200` 付费目标，建议至少 `2 台应用机 + 1 台托管数据库`

### 8.5 关键提醒（阿里云上最容易忽略）

- 带宽务必单独评估。语音场景若并发较高，`5 Mbps` 常常不够，建议从 `20~50 Mbps` 起评估。
- 监控必须有：CPU、内存、带宽、API 95/99 延迟、错误率、会话超时率。
- 只要出现晚高峰排队或超时，优先加机器与带宽，再优化代码。

|---|---:|---:|

- `3个月冲刺（推荐）`：覆盖完整备考周期，月均更划算

### 9.2 一级代理模式价格（你利润不变的做法）

采用“结算底价 + 代理加价”的单级模式：

- 你的结算底价（给代理）：
  - 标准版 1 个月：`58 元`
  - 冲刺版 1 个月：`97 元`
  - 冲刺版 3 个月：`254 元`
- 代理建议零售价（统一控价）：
  - 标准版 1 个月：`79 元`
  - 冲刺版 1 个月：`129 元`
  - 冲刺版 3 个月：`329 元`

这样做的效果：

- 你每单到手接近直销模型，不牺牲利润底盘
- 代理有清晰毛利空间（有动力推广）
- 价格体系可控（避免渠道乱价）

### 9.3 支付手续费（支付宝/微信）如何计入

不同主体、行业、通道（直连/服务商）费率会不同，建议按区间预算：

- 支付手续费预算：`0.6% ~ 1.2%`
- 再预留通道与对账损耗：`0.3% ~ 0.8%`
- 销售定价时统一按 `2%` 扣减测算（保守安全）

即：

- 实收收入估算 = `订单金额 × 98%`

示例（按 2% 计）：

- 59 元订单：实收约 `57.82 元`
- 99 元订单：实收约 `97.02 元`
- 259 元订单：实收约 `253.82 元`

### 9.4 支付集成实施建议（降低财务风险）

1. 支付宝与微信支付都接，避免单通道故障影响转化。
2. 统一订单中心，所有订单记录：`渠道/原价/优惠/实收/手续费/退款`。
3. 日对账自动化，异常订单 T+1 报警。
4. 渠道价与直销价分离，代理订单必须带代理 ID，便于结算与风控。
